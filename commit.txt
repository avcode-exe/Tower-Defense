Release 1.4.1 — Codebase Cleanup, Refactoring, Security, Performance, Tests, and Code Quality

A comprehensive 6-phase overhaul of the Tower Defense codebase. 7,018 lines removed, 1,691 added (net −5,327 lines). All 87 tests pass, lint clean.

BREAKING CHANGES:
- Removed legacy js/ directory (19 non-module files, ~6,300 lines). App loads exclusively from src/ via ES modules.
- Removed Web Worker (simWorker.js). Game loop now runs via main-thread requestAnimationFrame with fixed-timestep accumulator.

PHASE 1 — CLEANUP:
- Deleted js/ directory (19 legacy non-module JS files superseded by src/)
- Removed "js/**/*" from package.json files array, lint/format scripts
- Removed 37 vestigial globals from .eslintrc.json (all already resolved by ES module imports)
- Removed unused imports in main.js (UI, CONFIG, LAYOUT)
- Added missing import: MONSTER_SPECS in main.js
- Fixed typeof game !== 'undefined' bug → game !== null (4 occurrences)

PHASE 2 — REFACTOR:
- game.js: Extracted 3 DRY helpers (_findClosestMonsterNear, _applySlowToMonster, _addGold)
- game.js: step() split into 8 sub-methods (_stepWaveSpawning, _stepTroops, _stepProjectiles, _stepMonsters, _stepMonsterAttacks, _cleanupDead, _stepWaveCompletion, _stepPopups)
- game.js: onMouseDown (247 lines) split into 10 focused sub-handlers
- game.js: Replaced 3 duplicated tile-index lookups, 5 duplicated slow-application blocks, 3 gold-capped arithmetic blocks
- main.js: Extracted syncSettingsToMainProcess() helper (deduplicates 3 settings-save blocks)
- main.js: Removed redundant typeof checks now that imports are correct

PHASE 3 — SECURITY:
- Added Content-Security-Policy meta tag to index.html
- Added IPC input validation in electron-main.js (save-settings: object + 100KB, save-game: object + 1MB, load-game: 1MB size check, skip-update: string + 50 chars)
- Fixed onUpdateStatus memory leak in preload.js (now returns cleanup function)

PHASE 4 — PERFORMANCE:
- Removed Web Worker (simWorker.js deleted, gameRuntime.js rewritten with direct rAF loop)
- Added Projectile object pooling (avoids per-frame allocations from ranged troops)
- Canvas save/restore optimization (non-shielded troops skip ctx.save/restore)
- Monster shadows rendered as fillRect instead of arc (faster)
- Lazy Set allocation in Monster (only allocated for pass-mode monsters)

PHASE 5 — TESTS:
- Added Vitest with 6 test files, 87 tests covering: utils (27), grid (13), config (16), pathGenerator (8), projectile (7), persistence (16)
- Updated package.json test script to "vitest run"

PHASE 6 — CODE QUALITY:
- Created src/ui/toast.js as proper ES module (replaces window.showToast global)
- Updated gameRuntime.js to import showToast from toast.js
- Converted window._updateDevSpawnCounts to local function in main.js

Files changed: 32 files, −7,018 lines, +1,691 lines
