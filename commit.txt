Left sidebar flicker fix + shop/upgrade margin (v1.2.0 beta 2)

=== Shop card hover flicker fix ===

When the user scrolled the mouse wheel over the left shop, the
hover highlight on the troop cards flickered rapidly. Root cause:
mouse stays stationary during scroll while cards move under it,
so hitShop() returns a different card index every frame, and
updateHover() re-assigns hoveredShopIndex on each call, swapping
the card background between hover and normal states.

Fix: freeze the hover highlight while shopScrollY is changing.
Once scroll stops and the next frame has the same scrollY, the
guard releases and the highlight re-evaluates under the cursor.

Files changed (flicker fix):
- js/ui.js (UI object, updateHover):
  * Add _prevShopScrollY field next to shopScrollY
  * In updateHover, early-return when shopScrollY differs from
    _prevShopScrollY (syncs the field and skips hitShop)
  * When scroll is idle, the existing null-check and hitShop
    call proceed unchanged

=== Margin between shop cards and selected-troop UI ===

The bottom of the scrollable shop card list touched the top of
the selected-troop info panel with 0px gap. Added 6px of visual
breathing room.

Files changed (margin):
- js/ui.js (drawShop, areaBottom calculation):
  * Change `RENDERER.height - 204` to `RENDERER.height - 210`
    when a troop is selected, so the card area ends 6px above
    the info panel (which stays at height - 204)
  * The "no troop selected" branch is unchanged

=== Info panel width fix ===

The selected-troop info panel (Knight/DMG/SPD/RNG/HP/DPS) was
200px wide while the HEAL and Sell buttons below it were 224px
(UI_LAYOUT.SHOP_WIDTH - 16). Misaligned right edge looked ugly.
Widened the info panel to match.

Files changed (panel width):
- js/ui.js (drawShop, selected-troop info panel):
  * Both UIRoundRect calls (fill + stroke) changed from width
    200 to UI_LAYOUT.SHOP_WIDTH - 16 (224px)
  * Text x positions (18) unchanged — same 10px left padding
  * panelY (height - 204) and panelH (72) unchanged
