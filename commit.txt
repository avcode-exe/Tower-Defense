v1.3.0 — Bug fixes, performance improvements, and code cleanup

Bug fixes:
- Fix window resize crash (renderer.js didn't store canvas reference)
- Fix takeDamage returning wrong damage value when Ice Wizard shatter fires
- Fix NaN cached slow stats for non-Ice Wizard troops (null guards)
- Fix melee troops wasting cooldown when game is null
- Fix division-by-zero risk in getHpRatio/getHpPercent
- Fix const reassignment crash in tooltip (ui.js tipY)
- Fix wave manager custom wave format (objects → tuples)
- Fix cancel-update not actually preventing downloaded handler

Performance:
- Reuse splash hit buffer instead of allocating new array per call
- Hoist performance.now() before troop render loop (called once per frame)
- Remove duplicate _updateCardAreaBottom call in drawShop
- Replace JSON deep-clone in settings read with manual spread
- Cap notifications array at 50 entries

Code cleanup:
- Remove unused CONFIG constants (SHATTER_WINDOW, hud/hudText/hudAccent/selected/sell colors)
- Consolidate duplicate pre-release detection functions
- Remove redundant dSq===0 check in projectile.js
- Add card overflow mask to prevent scroll bleed into stats panel
- Improve maxed stat button visibility (background + border)

UI polish:
- Modern window margin (8px inset)
- Dynamic-width sidebar panels
- Gradient shop cards with type badges
- Improved shield shop design
