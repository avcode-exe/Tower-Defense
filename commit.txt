Release 1.5.0: Healer support troop

Compare against the 1.4.1 baseline, this release focuses on milestone 2 by introducing the Healer support troop and keeping the release branch in a clean, stable state for the 1.5.0 release.

Healer implementation:
- Adds the Healer as a support troop available from the shop.
- Targets the lowest-HP damaged ally instead of spending gold on additional attacks.
- Heals allies without spending gold, enabling sustained defense and ally preservation.
- Adds multi-target heal upgrades so Healer support can scale beyond a single ally.
- Adds green heal visuals and particles to make support healing clear during gameplay.
- Adds HPS/DPS helpers and placement/shop readouts so players can evaluate Healer value and placement.

Safe release-readiness fixes:
- Normalizes the stable release version to 1.5.0.
- Improves persistence validation and restores cleanup for safer save/load behavior.
- Fixes dev reset behavior so local development state resets predictably.
- Expands and updates tests and formatting to keep the release branch validated.

Swarm monster work is not included in this release and is planned for 1.5.1.

Validation passed:
- npm test
- npm run lint
- npm run format:check

GitNexus detect_changes was run before commit and reported critical risk due to many changed files and symbols. This is expected for the release commit and was authorized by the release scope.
