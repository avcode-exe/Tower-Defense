fix: screen shrinks when placing troops + clip range circles to gameplay area

Bug fix: placing a troop caused the entire canvas to visually shift/shrink.
The non-shielded troop render path called ctx.setTransform() +
RENDERER.applyMapTransform() which leaked a ctx.save() frame onto the
canvas state stack every frame per troop. The single restoreTransform()
at end-of-frame only popped one level, leaving the map transform still
active when UI overlay drawing began — rendering HUD, shop, and preview
panels at map-offset coordinates.

Fix: replaced setTransform + applyMapTransform with proper ctx.save() /
ctx.restore() for non-shielded troops, matching the shielded troop path
that already worked correctly. Zero save-frame leakage, balanced stack.

Also clip the troop range circle (both selected-troop and placement-ghost)
to the gameplay area bounded by the collapsible sidebars (shop left,
shield shop right, HUD top, preview bottom) so the dashed range indicator
no longer draws over UI panels. Clip rect respects collapsed sidebar state.

Files: src/rendering/gameRenderer.js, src/ui/placement.js
