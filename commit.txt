1.2.0 official release — refactoring, bug fixes, and performance improvements

This release stabilizes the 1.2.0-beta line with extensive code quality
improvements and critical bug fixes across the entire codebase.

Refactoring:
- Extract shared LAYOUT constants to config.js, eliminating hardcoded
  pixel coordinate duplication between ui.js and game.js
- Monster.js: eliminate duplicate position update code path in update()
- Troop.pickTarget: add bounds checks for edge grid tiles
- particles.js: replace string-key particle bucketing with numeric
  encoding for 300+ fewer string allocations per frame

Bug fixes (15 total across 9 files):
- game.js: fix CRITICAL ReferenceError in togglePopupEl (Alt+C/M/U
  shortcuts); fix worker fallback version guard preventing double-speed
  simulation after restart; fix confirmation flag stacking; add break
  after DEFEAT state to prevent duplicate leak sounds
- monster.js: fix findTarget() null check on empty tiles; replace
  shared mutable _damageResult with fresh object literals per call
- ui.js: fix inverted toggle arrow directions; fix shield shop
  collapse direction; add tooltip vertical bounds clamping; fix
  tooltip text overflowing background on narrow screens
- renderer.js: add window resize handler for proper canvas scaling
- input.js: fix removeEventListener passive option mismatch
- projectile.js: wrap onImpact in try-catch to prevent zombie
  projectiles on callback throw
- particles.js: clear() now fully resets color index state
- waveManager.js: custom wave preview now reflects actual queue
- grid.js: add OOB warning when returning BLOCKED
