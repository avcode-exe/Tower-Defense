1.3.0-beta.1 — Ice Wizard + Bug Fixes

New Features:
- **Ice Wizard** (hotkey 0): 10th troop type, ranged attacker with splash
  that applies slow on hit. Cost: 200g, Base HP: 60, Damage: 6, Range: 3,
  Attack Speed: 1.4s, **Splash: 1.5 tiles**
- **Slow Status Effect**: Monsters hit by Ice Wizard are slowed to 50% movement
  speed for 2.5 seconds. Visual indicator: icy blue overlay on slowed monsters
  + particle burst on slow application.
- **Shatter Mechanic**: The next hit on a slowed monster deals +50% bonus
  damage. Re-applying slow refreshes duration but does NOT re-arm shatter.
- **Shielded Immunity**: Monsters with active shield are immune to slow until
  shield breaks.
- **Splash Applies Slow**: All monsters hit by splash take damage AND are slowed.
- **5th Upgrade Stat (SLW)**: Ice Wizard has a 4th upgradable stat — SLW
  increases slow potency (factor), duration, and shatter bonus per level:
  - L1: 50% speed, 2.5s, +50% dmg
  - L5: 26% speed (74% slow), 5.2s, +143% dmg

Balance:
- Added 4 new CONFIG constants for slow/shatter scaling:
  SLOW_FACTOR_SCALE_PER_LEVEL=0.85, SLOW_DURATION_SCALE_PER_LEVEL=1.2,
  SHATTER_BONUS_SCALE_PER_LEVEL=1.3, SHATTER_WINDOW=3.0s
- Upgrade cost follows existing formula (base × 1.35^(level-1))

UI Fixes:
- **SLW upgrade button click handler fixed** — was missing 'slow' from stats array
- **Dynamic troop stats panel** — auto-sizes based on visible stat lines (no text overlap)
- **Dynamic shop card area** — bottom boundary adjusts per troop (Ice Wizard's extra SLW line no longer blocks shop cards)
- **Load progress dialog** — shows popup asking to load previous progress or start new game

Critical Bug Fixes:
- **Troop constructor now calls `_recomputeStats()`** — cached stats (maxHp, slowFactor, etc.) now properly initialized at spawn instead of waiting for first upgrade
- **Save/Restore includes `slowLevel`** — Ice Wizard slow upgrades now persist across save/load (was lost on reload)
- **`getTotalInvested()` includes `slow` upgrades** — sell value now accounts for slow upgrades correctly
- **Shield monster immunity** — shielded monsters immune to slow while shield active (already implemented)

Visual:
- Slowed monsters render with semi-transparent icy blue overlay (#7fdbff @ 50%)
- Particle burst on slow application (ice blue particles)
- All splash targets get slow applied + particle effect
