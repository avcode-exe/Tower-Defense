Comprehensive refactor, bug fixes, and quality improvements across entire codebase

6-agent parallel overhaul: 25 files changed, −604 lines, +408 lines. Lint clean, 87 tests pass.

BUG FIXES:

- game.js: Fix ranged projectile crash — `CONFIG.PROJECTILE_STYLES` was undefined because
  PROJECTILE_STYLES is a separate named export from config.js, not a CONFIG property.
  Added missing import and used `PROJECTILE_STYLES` directly.
- game.js: Fix non-shielded troop render leak — `setTransform` + `applyMapTransform` leaked
  a ctx.save() frame per troop per frame, corrupting the canvas state stack and causing the
  entire screen to visually shift after placing any troop. Replaced with balanced save/restore.
- game.js: Fix misleading defensive kill — `_stepMonsters` called `damageMonster(m, m.maxHp)`
  when `m.hp <= 0`, which routed through the full damage pipeline only to hit an early-return
  guard. Replaced with direct `m.alive = false`.
- main.js: Fix `updateDevSpawnCounts` scoping — inner `const` shadowed outer `let`, so the
  Dev panel Reset button called a no-op and counts never refreshed. Removed `const`.
- main.js: Fix `showPopup` double-fire — both `transitionend` listener and `setTimeout`
  fallback could call `openTarget()`. Added `clearTimeout` in the onDone handler.
- gamePersistence.js: Add missing `speed` field validation in `isValid()` — non-number or
  negative speed would pass validation and break the game loop.
- gamePersistence.js: Add missing troop `hp` field validation in `isValid()` — non-numeric
  hp would pass validation but break `Math.min` in `apply`.
- electron-main.js: Fix `delete-save` handler missing return value.
- gameRenderer.js: Fix cursor crash when `RENDERER.canvas` is null.
- rendering/renderer.js: Batch grid line drawing into single beginPath/stroke instead of
  32+ separate pairs, reducing draw call overhead.
