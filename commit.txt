Release v1.4.1 — Final bug fixes, optimizations, and code quality polish

BUG FIXES:
- Fix projectile crash: CONFIG.PROJECTILE_STYLES was undefined because PROJECTILE_STYLES
  is a separate named export from config.js, not a CONFIG property
- Fix non-shielded troop render canvas state stack leak causing screen shift on placement
- Fix updateDevSpawnCounts scoping bug where inner const shadowed outer let, breaking
  the Dev panel Reset button
- Fix showPopup double-fire where both transitionend and setTimeout could call openTarget
- Fix save-settings/save-game JSON size mismatch: size checks used compact JSON but
  wrote pretty-printed, silently exceeding the cap on disk and rejecting on load
- Fix chainSpark() returning shared template object instead of copy, corrupting template
  for all subsequent calls
- Fix autoDownload default mismatch between renderer (true) and main process (false)
- Fix stale hardcoded version 1.3.0 in updateManager defaults
- Fix delete-save handler missing return value
- Fix save-settings size check to account for pretty-printed JSON expansion
- Fix monster shadow rendering: batched arcs in single beginPath created dark filled
  regions between shadow circles via implicit canvas path connections

PERFORMANCE:
- Batched grid line rendering into single beginPath/stroke (eliminated 32+ draw calls)
- Replaced particle config object spread allocations with shared temp config object
- Particle draw loop: replaced delete with null assign (avoids V8 deopt), simplified
  alpha math, used bitwise integer truncation
- _getColorIndex: direct property access instead of in operator (avoids prototype walk)
- _cleanupDead: inline compaction tracking instead of indexOf() O(n) scan
- Cached CONFIG.TILE_SIZE, GRID_SIZE, TILE_BUFFER in hot loops
- Eliminated redundant save/restore/translate/scale in drawStaticLayers

REFACTORING:
- Extract _applyHitAtPosition consolidating duplicated chain/splash/single-hit logic
- Flatten nesting with guard clauses throughout game.js
- Troop upgrade/cost/maxed methods refactored from if-chains to map lookups
- Audio methods: extracted _toneRamp helper, added missing _enabled guards
- Monster: null-safety fallback for unknown spec levels
- UI: fillStrokeRoundedRect helper reducing duplicated beginPath/fill/stroke patterns

SECURITY:
- Added regex validation for skip-update version strings
- Added typeof+allowlist validation for set-update-channel parameter
- Upgraded CSP with connect-src and worker-src directives
- Added ARIA labels/roles to buttons, dialogs, inputs, and SVG icons

CODE QUALITY:
- Prettier formatting applied across all source files
- Import reordering for consistency across all modules
- Removed dead CSS rules, unused variables, stale defaults
- Updated README to v1.4.1 with ES modules architecture and test suite docs
