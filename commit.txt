Code quality overhaul: security fix, dead code removal, performance improvements, and linting setup

Security:
- Fixed XSS vulnerability in toast notifications (main.js) — replaced innerHTML
  with safe appendChild + createTextNode to prevent HTML injection from update
  version strings

Bug fixes:
- Fixed formatting bug in game.js step() — two statements on one line with
  inconsistent whitespace (AUDIO.monsterLeak() and if block on same line)
- Fixed inconsistent indentation in Game.onKeyDown() — Alt+key popup toggle
  block was dedented outside the method scope

Dead code removal:
- Removed unused _buildPathSegments() method from Game (duplicated logic in
  GameWorldFactory.createFresh)
- Removed unused drawDevRightPanel() from UI (~80 lines, DEV panel moved to
  bottom-bar popups in main.js)
- Removed unused DEV_STARTING_GOLD constant from config (code uses Infinity)

Performance:
- Replaced O(n) linear troop scan in canPlace() with O(1) tile-index lookup
  using the existing _troopTileIndex spatial index
- Eliminated createLinearGradient() calls per card per frame in UI shop and
  shield shop — replaced with solid fills (gradient over 58px is imperceptible),
  removing 10+ gradient allocations per frame

Config centralization:
- Added CONFIG.MELEE_DAMAGE_REDUCTION (0.3) — replaces hardcoded 30% in
  Game.damageTroop()
- Added CONFIG.TROOP_HEAL_HP_RATIO (0.1) — heal amount percentage
- Added CONFIG.TROOP_HEAL_COST_RATIO (0.1) — heal cost percentage
- Updated Troop.getHealCost(), Troop.heal(), and UI heal display to use
  new CONFIG constants

Error handling:
- Added try/catch error boundary around Game._runSimTick() to render a crash
  screen instead of white-screening on unhandled exceptions

Tooling:
- Added ESLint with .eslintrc.json configured for script-tag architecture
  (all globals declared, browser+worker+node envs, file overrides)
- Added Prettier with .prettierrc.json (single quotes, trailing commas,
  2-space indent, 120 char width)
- Added npm run lint, npm run lint:fix, npm run format, npm run test scripts
- Added eslint and prettier as devDependencies
- ESLint passes clean: 0 errors, 0 warnings

Docs:
- Updated README.md: added ESLint and Prettier to Tech Stack, added
  lint/format commands to Building section
