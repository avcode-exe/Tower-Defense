Release 1.4.1-beta.2 — Documented Bug Fixes, Lint Hardening, and Production Release Prep

Summary
- Bumped the application version from 1.4.1-beta.1 to 1.4.1-beta.2.
- Resolved all currently documented application issues before release.
- Hardened linting, input lifecycle handling, projectile combat state, save/load validation, and defeat-state handling.
- Prepared the project for a GitHub-published Windows Electron build using tag 1.4.1-beta.2.

Release Context
- Release version: 1.4.1-beta.2
- Git tag: 1.4.1-beta.2
- Target platform: Windows x64 NSIS installer
- Publisher: GitHub Releases under avcode-exe/Tower-Defense
- Build command: npm run release
- Expected uploaded artifacts: Tower Defense Windows executable and electron-builder latest.yml metadata

Core Application Fixes
- Fixed ESLint ES module parsing by changing parserOptions.sourceType from script to module, allowing src/**/*.js modules to lint successfully.
- Removed the remaining no-constant-condition lint warnings by replacing infinite while loops in both src/game.js and legacy js/game.js with explicit popDueMonster() loops.
- Fixed input listener cleanup by storing reusable listener option objects and removing wheel listeners with the same { passive: false } options used when adding them.
- Updated mouse hover coordinate calculation to read the current canvas bounding rect on every mousemove, avoiding stale hover coordinates after layout changes.
- Cleared reusable projectile buffers during fresh game setup and snapshot restoration to prevent stale monster references after restart or load.
- Prevented slow effects from being applied to monsters killed by splash damage.
- Prevented slow effects from being skipped when a direct projectile's original target dies before impact by applying slow to the replacement closest alive monster when appropriate.
- Hardened save data by serializing Infinity dev gold/lives as null and restoring them as Infinity for dev mode.
- Added SaveSerializer.isValid() validation before loading saves, causing malformed or partial saves to be ignored instead of restoring invalid state.
- Reset transient UI/runtime confirmation and cooldown state when restoring a save.
- Stopped simulation progression immediately after defeat is applied.
- Improved keyboard input handling by returning after Escape, Space, and Enter handling, preventing Enter from falling through into Alt shortcut handling.
