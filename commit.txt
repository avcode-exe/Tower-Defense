Release 1.4.1-beta.1 — ES Module Refactor, Bug Fixes, and Production Hardening

Summary
- Converted the entire codebase from 18 inline script-tag files to a modern ES module architecture
- Fixed runtime errors that caused blank screen and tooltip rendering failures
- Disabled automatic DevTools auto-open on Electron startup
- Bumped version from 1.4.0 to 1.4.1-beta.1 and updated Electron Builder configuration

Codebase Modularization
- Replaced 18 <script src="js/..."> tags in index.html with a single <script type="module" src="src/main.js">
- Created 27 ES modules under src/ with explicit import/export dependency graph
- Extracted Game.render() into standalone renderGame() and updateCursor() in src/rendering/gameRenderer.js
- Split monolithic ui.js (1388 lines) into 9 focused sub-modules under src/ui/
- Preserved backward compatibility by exposing window.game, window.showToast, and window.electron globals
- Added src/**/* to Electron Builder files array so packaged app includes all modules

Bug Fixes
- Fixed ReferenceError: CONFIG is not defined in src/utils.js by adding missing import { CONFIG } from './config.js'
- Fixed TypeError: this._drawShopTooltip is not a function by binding _drawShopTooltip and _wrapText onto the UI object in src/ui/index.js
- Fixed stale game.render() call in src/gameRuntime.js pause-render loop; replaced with renderGame(game) and added updateCursor(game)
- Removed debug openDevTools call from electron-main.js so DevTools no longer auto-open on game launch

Module Structure
- src/main.js — Application entry point, canvas initialization, save/load wiring
- src/game.js — Core Game orchestrator, simulation tick loop, pause/render coordination
- src/rendering/gameRenderer.js — All canvas rendering: map, troops, monsters, projectiles, particles, HUD, shop
- src/ui/index.js — Barrel export assembling unified UI namespace from 9 sub-modules
- src/ui/shop.js, hud.js, shieldShop.js, preview.js, placement.js, overlays.js, utils.js, constants.js
- src/config.js — Centralized CONFIG, LAYOUT, monster/troop specs, waves, projectile styles
- src/audio.js — AudioManager with mute and volume controls
- src/updateManager.js, input.js, gamePersistence.js, gameRuntime.js, grid.js, pathGenerator.js, monster.js, troop.js, projectile.js, waveManager.js, particles.js, utils.js, simWorker.js

Verification
- All 28 module files pass Node.js syntax validation
- Zero missing import paths confirmed via dependency verification
- Game launches without console errors in Electron
- Canvas renders correctly and game is playable end-to-end

Breaking Changes
- None. The bundled renderer and preserved window globals keep existing integrations working.

Files Changed
- Modified: index.html, package.json, electron-main.js
- Added: src/ (27 module files)
- Removed: js/ (18 legacy script files no longer referenced)
- Note: Generated build artifacts exe and latest.yaml are not tracked in source control
