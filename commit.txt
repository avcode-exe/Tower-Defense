Balance and dev-mode economy updates

- Movement speed categories:
  - Added CONFIG.MOVEMENT_SPEED_CATEGORIES and CONFIG.MOVEMENT_SPEEDS.
  - Mapped monsters to categories and numeric speeds:
    - Grunt: medium, speed 1.0
    - Runner: very fast, speed 3.0
    - Brute: slow, speed 0.8
    - Elite: medium, speed 1.0
    - Champion: slow, speed 0.8
    - Boss: very slow, speed 0.6
    - Shielded: medium, speed 1.0
    - Spear: fast, speed 2.0
    - Necromancer: slow, speed 0.8
- Economy changes:
  - Monster defeat now awards base reward + 1 gold using _addGold.
  - Healer troop cost reduced from 150 to 140.
- Dev-mode economy:
  - Dev mode placement, upgrades, heals, and shields are free and bypass gold checks.
  - _addGold keeps dev mode gold at Infinity.
  - Shop cards are visually affordable in dev mode.
  - Normal mode still requires and spends gold normally.
- Splitting change:
  - Splitting monsters never produce Runner children.
  - Brute splits into Grunts, skipping Runner.
  - Elite and Champion split normally one level lower.
  - Runner still does not split.
- Tests:
  - Added tests/config movement speed coverage.
  - Added tests/devMode.test.js.
  - Added tests/splitting.test.js.
  - Updated Healer, Necromancer, and splitting reward expectations.
- Validation:
  - npm test passed with 16 files and 188 tests.
  - npm run lint passed.
  - npm run format:check still fails only on src/gamePersistence.js, src/ui/preview.js, and src/updateManager.js.
