v1.4.0 — Spear monster, Game class refactor, code quality overhaul, and code formatting

This release consolidates all v1.4.0-beta.1 through beta.4 features, code quality
improvements, and applies consistent code formatting across the entire codebase.

Version bump:
- package.json, package-lock.json, gamePersistence.js save version
  bumped from 1.4.0-beta.4 to 1.4.0

Code formatting (Prettier):
- Applied Prettier formatting to all JS and CSS files for consistent style
- 2-space indentation, single quotes, trailing commas, 120 char line width

New features (since v1.3.0):
- Spear monster (level X) — new monster type that slows troops to half speed
  while attacking from a 2.5-tile radius, attacks while moving
- Game class decomposition — extracted GameRuntimeController (Web Worker
  lifecycle, pause/resume, resize) and persistence layer (SaveSerializer,
  GameWorldFactory, GameSnapshotRestorer)
- Animated tray windows — smooth roll-up/down transitions for bottom bar popups,
  only one popup open at a time
- Smart cursor — standard arrow by default, hand pointer on clickable elements
- Monster info panel — bottom bar popup showing all monster stats
- Dev mode improvements — dedicated DEV popup with per-monster count controls
- Mute/unmute button — toggle audio with volume save/restore
- Monster attack mode rework — three distinct modes: stop, pass (penetration),
  slow (slows near troops)

Monster attack modes:
- stop: Stops moving when a troop is in range, attacks, then resumes (default)
- pass: Always moves at full speed, deals damage while passing (Runner, Spear)
- slow: Slows to half speed near troops, attacks closest (Spear)

Code quality improvements:
- Fixed XSS vulnerability in toast notifications (innerHTML → createTextNode)
- Removed dead code (~100 lines): drawDevRightPanel, _buildPathSegments,
  unused DEV_STARTING_GOLD constant
- Added error boundary for sim tick to render crash screen on exceptions
- Extracted magic numbers into CONFIG constants (MELEE_DAMAGE_REDUCTION,
  TROOP_HEAL_HP_RATIO, TROOP_HEAL_COST_RATIO)
- Replaced O(n) canPlace() scan with O(1) tile-index lookup
- Eliminated per-frame gradient allocations in UI shop cards
- Added ESLint and Prettier tooling with npm scripts

Bug fixes:
- Fixed reset button hitbox (w:36 → w:50 to match visual label)
- Fixed formatting bug in step() method
- Fixed inconsistent indentation in onKeyDown

Files changed:
- package.json, package-lock.json: Version 1.4.0, added eslint + prettier
- gamePersistence.js: Save version bumped to 1.4.0
- game.js: Refactored lifecycle, error boundary, canPlace optimization,
  dead code removal, formatting bug fix, indentation fix
- ui.js: Animated tray windows, dev panel, monster info, dead code removal,
  gradient cache optimization
- main.js: Bottom bar popups, monster info, DEV panel, XSS fix
- config.js: Added new layout constants, removed unused DEV_STARTING_GOLD,
  added MELEE_DAMAGE_REDUCTION and heal ratio constants
- monster.js: Spear monster, attack mode rework (stop/pass/slow)
- audio.js: Mute/unmute button support
- renderer.js: Cache management improvements
- input.js: No changes
- css/style.css: Tray window animations, panel styling
- index.html: Updated script tags, bottom bar button structure
- icon.png, icon.svg: Redesigned tower icon with glowing beacon
- .eslintrc.json, .prettierrc.json: New — code quality tooling
- README.md: Updated tech stack, controls, troop/monster tables, build commands
