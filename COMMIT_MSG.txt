feat(ui): add zoom controls, 2-line HUD fallback, auto-collapse sidebars, and settings panel rework

This commit bundles several interdependent UI/UX improvements for the v1.7.0
beta release. Changes span the zoom system, HUD layout, sidebar management,
and settings panel.

─── Zoom Controls ───

  Add keyboard zoom with Ctrl+/-/0 (range 1.0x–2.0x). The underlying
  RENDERER now carries a `zoom` property; applyMapTransform multiplies the
  map scale by an "effective zoom" (capped at 1x when panels crowd the
  canvas) so the enlarged map expands into empty space rather than
  overflowing into sidebars.

  A zoom indicator pill appears in the bottom-right corner with a
  fade-in/hold/fade-out animation whenever the user changes zoom. The
  indicator pulses red when zoom is at either endpoint (100% or 200%).

  The UI layout system gained `UI_LAYOUT._zoom` and the `LAYOUT_ZOOM`
  proxy. A new `zoomFont(c, px, weight)` utility in `ui/utils.js`
  replaces hardcoded font sizes across all UI panels (shop cards,
  shield shop, preview, placement, overlays) — every `zp()` call in
  these files now scales pixel offsets proportionally with zoom.

─── 2-Line HUD Fallback ───

  When zoom pushes the top HUD elements beyond the available width, the
  speed label and speed buttons drop to a compact second row (~30px)
  below the main 56px row.  The start/pause/resume (ctrl) button stays
  on row 1 so it remains accessible.

  Before the two-line cutover, a dynamic gap-compression pass shrinks
  the gaps between gold/lives/wave/reset proportionally (minimum 10%
  of original gap).  If compression alone cannot prevent overlap, the
  speed section moves to row 2 and row 1 reverts to its natural
  zoom-scaled spacing (not capped at zoom=1 values, giving a balanced
  look).

  Click handlers in `game.js` and cursor hit-testing in
  `gameRenderer.js` both check `UI._hudTwoLines` and use stored
  compact button dimensions (`_speedBtnOffsetY`, `_speedBtnGap`,
  `_speedBtnH`) stored by `drawHUD()`.

─── Auto-Collapse Sidebars ───

  When the rendered map is narrower than either the left (shop) or
  right (shield shop) sidebar, both sidebars automatically collapse to
  20px-wide bars.  Restoration uses the "expanded map width" — what
  the rendered map would be with full-size sidebars — rather than the
  current collapsed map width.  This prevents flip-flop because the
  expanded width stays narrow (computed with full sidebar sizes) after
  auto-collapse.

  The restore path is gated on a 1.15x hysteresis threshold and also
  handles zoom-out: when the user zooms out and the window has room
  for full sidebars, they are restored.  Manual toggle detection
  compares against the literal `true` (expected auto-collapse state)
  rather than `_prevCollapsedShop` (pre-collapse state) to avoid false
  resets on every frame.

  Auto-collapse is checked in `RENDERER.resize()` (on window resize or
  panel toggle) and in `renderGame()` (on zoom change).

─── Settings Panel Rework ───

  The settings popup switched from a top-tabbed layout to a
  sidebar + panel structure.  The "Game" tab (starting gold, lives,
  max wave, speed default) was removed entirely.  Audio controls were
  reorganised into labelled rows with inline mute checkboxes.

  Electron main (`electron-main.js`) sanitisation removed the `game`
  settings section and the `fontSizeScale` accessibility field.
  Defaults in `settingsDefaults.js` were updated accordingly.  The
  in-game `scrollZoom` toggle was relabelled to "Keyboard Zoom" to
  reflect that it controls the Ctrl+/- shortcut rather than actual
  scroll-wheel zoom.

  `LAYOUT_ZOOM` export.

─── Known Limitations ───

  • The auto-collapse can override a manual toggle when the window is
    too narrow for expanded sidebars — the system re-collapses on the
    same frame.
  • Zoom-induced layout recomputation calls `RENDERER.resize()` during
    the render loop, which clears and resets the canvas; the
    subsequent `beginFrame()` re-fills it, so no visible glitch occurs
    on screen.
